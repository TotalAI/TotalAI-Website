How To Create an InputConditionType (ICT)

Start by Searching Existing InputConditionTypes

Take a look at the ICTs that ship with TAI. See if there is one that does what you want or is at least similar. To see the full list with descriptions go here.

Starting ICT Template

Your ICT needs to inherit from ICT and override only one function: 


  class YourICT : InputConditionType
  {
    // Any Variables that should be set on a per ScriptableObject Asset basis

    public override bool Check(Agent agent, InputCondition inputCondition,
                               OutputChange outputChange)
    {
      // Return true if the agent passes this ICT else return false      
      if (...)
        return true;
      return false;
    }
  }


Will your ICT be "Fixable"?

This will be used if an Agent is using a PlannerType that allows chaining and therefore creates Plan Trees. For example the GOAPPlannerType does this by using chaining, connecting a failed IC to an OC that fixes the IC, to create child Mappings.

If you want your ICT to be able to chain with an OC do the following:

  • Create the OCT that can fix your ICT. See Creating an OCT
  • Select the OCT ScriptableObject Asset and add your ICT to the OCT's MatchOTCs list.
  • Add the Matching logic in the OCT's Match method.
  • Click on the GOAPPlannerManager and make sure you see your IC matched to a MappingType with the matching OTC.
